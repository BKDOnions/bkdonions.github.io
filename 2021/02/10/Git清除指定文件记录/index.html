Git清除指定文件记录

本文参考源：Mr.True@csdn Github Official Docs

起因是自己误操作将Intellij Idea的配置文件commit到git记录上面，考虑到部分配置涉及个人隐私，所以想清除掉指定文件的记录

建议在删除记录之前先将需要保存在本地但又不想commit到git记录的文件先备份，在关键操作完成后再将文件覆盖或者选择删除备份

第一步：确定敏感文件

敏感文件为Initial commit中的.idea、target和*.iml

相对于项目git记录根目录为：

./.idea

./target

./*.iml

第二步：清除相关记录

通过git filter-branch清除相关文件记录并且重写所有tags

此处我使用的命令

第三步：强制覆盖记录

第四步：解除对本地存储库中的所有对象的引用和垃圾收集

确保没有什么问题之后,强制解除对本地存储库中的所有对象的引用和垃圾收集

至此已经完全清除相关记录了
